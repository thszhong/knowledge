架构的分类
- RUP 4+1： 场景视图、逻辑视图（功能需求）、开发视图、处理视图、物理视图

架构的衡量
- 业务角度：解决当前业务问题、高效支撑中短期业务需求、前瞻性考虑未来业务发展
- 技术视角：稳定性、文档化、可拓展性、安全、技术先进性
- 管理视角：成本、团队成长
